\section{Introduction and Related Work}

The fashion industry nowadays relies on intelligent systems that analyze clothing items from images for applications such as online retail search, recommendation systems and trend analysis. Manual product annotation is time-consuming, costly, and often inconsistent, which motivates automated approaches for clothing recognition and attribute prediction.

CNN-based methods have achieved strong results on clothing classification~\cite{bossard_2012}, detection~\cite{alamsyah_2019}, and landmark localization~\cite{wang_2018}.

Jointly predicting multiple attributes remains challenging: the same garment can look very different depending on pose and context, category boundaries are visually ambiguous (e.g., skirts vs. short dresses), and attributes such as material or shape rely on different visual cues than category identity. Amin et al.~\cite{amin2023fsap} had to construct a dedicated 300k-image dataset with 30 domain experts to capture sufficient attribute-level labels.

The task can be described as follows: Given an input image, the pipeline (i) detects fashion items, (ii) classifies the item type (e.g., shirt, dress, trousers), and (iii) predicts fine-grained attributes such as pattern, sleeve length, neckline, fabric, or fit. Each stage is evaluated separately: detection by mAP and mAR at IoU\,$\geq$\,0.75; classification by accuracy and coverage over a held-out test set; and attribute prediction by per-head accuracy and macro F1 across the six attribute groups.

\section{Method}
\label{sec:method}

\subsection{Model Architecture}
The pipeline uses a dedicated model at each stage: object detection first localizes clothing items, then cropped regions are passed to the classification and attribute prediction models. Table~\ref{tab:models} (Appendix~\ref{app:tables}) summarizes all models investigated.


\paragraph{Object Detection}
Object detection was performed using a Faster R-CNN ResNet50 FPN v2 model pre-trained on the COCO dataset and fine-tuned on DeepFashion2. The model uses a convolutional backbone for feature extraction and a Region Proposal Network (RPN) to generate candidate object regions. During inference, high-confidence proposals are retained while overlapping detections are suppressed, to yield the final set of bounding boxes.

Mean Average Precision (mAP) and Mean Average Recall (mAR) are computed with torchmetrics at an Intersection over Union (IoU) threshold of 0.75.
Precision ($= TP/(TP{+}FP)$) and Recall ($= TP/(TP{+}FN)$) measure the proportion of correct box predictions and the proportion of ground-truth objects detected, respectively.
Two boxes are considered a match if their overlap is at least 75$\%$. This strict threshold was chosen to only count precise localizations as correct. As shown in Figure~\ref{fig:train_res_obj_detection} (Appendix~\ref{app:figures}), the box loss is low; however, the mAP remains relatively low, which is attributed to the large number of generated boxes per object.

\paragraph{Classification: Custom CNN}
In a first approach a custom CNN was developed and trained from scratch in a residual learning paradigm~\cite{DBLP:journals/corr/HeZRS15}: each block combines a convolutional transformation path with a skip connection. Each block contains three consecutive $3\times3$ convolutional layers with Batch Normalization and ReLU activation, where the final convolution is applied without activation before residual addition: $y = \mathrm{ReLU}(F(x) + S(x))$. The architecture comprises nine residual blocks with progressively deeper channels and spatial downsampling that reduce the input from $224\times224$ to $7\times7$. The classification head applies adaptive average pooling, dropout ($p=0.4$), and a fully connected output layer.

\paragraph{Classification: Transfer Learning with EfficientNet}
Since the training results obtained with the custom CNN were not fully satisfactory (see Figure \ref{fig:train_ressults_res_cnn}), a transfer learning approach was explored. Pretrained EfficientNet-B1 and EfficientNet-B2 \cite{tan2020efficientnetrethinkingmodelscaling} were selected as backbone networks. EfficientNet applies compound scaling (a joint adjustment of depth, width, and resolution) for high accuracy with strong parameter efficiency. Both models were fine-tuned with the classification head replaced by a task-specific fully connected layer.

\paragraph{Attribute Prediction}
The same transfer learning strategy was applied for attribute prediction, but with a multi-head design: a shared pretrained backbone extracts a single feature vector from the input crop, and six parallel fully connected heads each independently predict one attribute group: texture (7), sleeve length (3), garment length (3), neckline (4), fabric (6), and fit (3), which covers all 26 dimensions of the DeepFashion attribute vector. Three backbone variants were compared, as summarized in Table~\ref{tab:attr_backbones}.

\subsection{Training}
\paragraph{Data and Preprocessing}
Two complementary datasets are used (Table~\ref{tab:datasets}). DeepFashion2~\cite{ge2019deepfashion2} provides 100K images across 13 categories with bounding box and category annotations, including both commercial and real consumer photos. Since it lacks attribute labels, attribute prediction uses a 20{,}000-image subset from DeepFashion~\cite{liu2016deepfashion} that covers 26 attributes across six groups. Training sets were class-balanced and all images scaled to $224\times224$ pixels. Attribute prediction crops were extracted using ground-truth bounding boxes and augmented with horizontal flips and color jitter; classification training used similar augmentations with additional rotations.

\paragraph{Loss and Optimization}
Classification models were trained with CrossEntropyLoss, while the detection model employed the standard Faster R-CNN objective. For attribute prediction, the loss is $\mathcal{L} = \sum_{g} w_g \cdot \mathrm{CE}(\hat{y}_g, y_g)$, where $w_g$ is an optional per-head weight. Per-stage optimizer settings are summarized in Table~\ref{tab:hyperparams}.

\paragraph{Category Classification}
For the EfficientNet-based models, a staged fine-tuning strategy was applied: the backbone was first frozen and only the classification head was trained, after which parts of the backbone were progressively unfrozen and jointly trained with the classifier. Empirically, this gradual unfreezing strategy led to improved test accuracy. The EfficientNet models were trained on images cropped according to ground-truth bounding boxes, ensuring that each training image contained only a single clothing category. Additional regularization was introduced through dropout layers in the classification head, while batch normalization within the custom CNN stabilized gradient flow and improved convergence.

\paragraph{Attribute Prediction}
The DeepFashion attribute subset is severely imbalanced; for example, the texture class \emph{solid} accounts for 47.5\% of training samples while \emph{lattice} accounts for only 1.7\%. To address this, per-class weights were computed within each group as $w_c = \frac{N}{K_g \cdot n_c}$ and applied to the cross-entropy loss. The weighted strategy achieved a slightly lower overall validation accuracy (0.8502 versus 0.8647 for the baseline), but improved performance on minority classes; for example, fit class \emph{conventional} improved from 13.4\% to 38.1\%. The weighted EfficientNet-B0 checkpoint was selected as the final model. Table~\ref{tab:attr_backbone_comparison} summarises the backbone comparison.



\section{Results}

\paragraph{Object Detection}
As shown in Figure~\ref{fig:train_res_obj_detection} (Appendix~\ref{app:figures}), box regression loss converges steadily over training. Boxes with a confidence score below 0.5 are first removed; NMS is then applied to suppress overlapping detections. Each surviving bounding box is cropped and forwarded to the classification stage; Figure~\ref{fig:boxes_filtering_comparison} (Appendix~\ref{app:figures}) illustrates the effect of this filtering.

Sleeve and length are predicted most reliably (accuracy $>0.93$), while fabric and texture are the weakest heads (macro F1 $0.663$/$0.699$), driven by visually ambiguous minority classes such as \emph{denim} and \emph{pleated}. The gap between macro and weighted F1 confirms minority class performance as the primary limitation.

\paragraph{Full Pipeline}
End-to-end attribute quality is assessed qualitatively since DeepFashion2 provides no attribute annotations. Figure~\ref{fig:pipeline_demo} (Appendix~\ref{app:figures}) shows a successful example with correct category and attribute predictions. Figures~\ref{fig:pipeline_failure_1} and~\ref{fig:pipeline_failure_2} illustrate a recurring failure mode: duplicate boxes for the same garment, which produced inconsistent attribute predictions across nearly identical crops (e.g. \emph{long sleeve} vs \emph{sleeveless} for the same garment).

\section{Conclusion}

This project implemented and evaluated a three-stage pipeline for fashion item detection, category classification, and attribute prediction. The best-performing model achieved 33.42\% exact match and 69.62\% coverage on the pipeline evaluation, and the attribute model reached a mean accuracy of 84.4\% across six attribute groups. Staged fine-tuning and weighted loss for class imbalance both proved effective: the weighted strategy notably improved minority class performance, e.g. the fit class \emph{conventional} improved from 13.4\% to 38.1\%.

A key insight was that the bottleneck in the full pipeline lies in classification rather than detection: box localization was reliable and the NMS filter delivered clean crops, but poor classification produced conflicting labels across duplicate boxes and compounded errors downstream. Image quality variation across the dataset (lighting, contrast, background clutter) also made generalization harder than anticipated, and stronger backbones tended to overfit.

For future work, an improvement would be to evaluate the attribute model on detector-predicted crops rather than ground-truth boxes, to obtain a true end-to-end accuracy measure.
