\section{Introduction}

The automatic understanding of visual content has become one of the central challenges in modern computer vision and computer vision applications. In the current document we will describe a computer vision problem in the field of fashion industry, which we aim to solve by using deep learning methods.
In particular, the fashion industry increasingly relies on intelligent systems capable of analyzing clothing-and fashion items from images for applications such as online retail search, recommendation systems, inventory management, and trend analysis. Manual annotation of fashion products is time-consuming, costly, and prone to inconsistency, motivating the development of automated methods that can recognize fashion items and describe their visual attributes and characteristics.

The visual analysis and classification of clothing has gained increasing importance in the context of e-commerce and social media ~\cite{wang_2018, alamsyah_2019}.
A comprehensive analysis of clothing involves several sub-tasks, including landmark detection, classification, and semantic segmentation in real-world scenes.

\textbf{Related Work.}
A classical machine learning approach is presented in ~\cite{bossard_2012}.
The work describes a pipeline for clothing classification based on learning visual attributes. These attributes are used to represent the visual properties of garments.
For classification, a Random Forest is employed, combined with multiple Support Vector Machines (SVMs) for attribute learning.

More recent work mainly focuses on CNN-based methods ~\cite{alamsyah_2019, maryamah_2023, wang_2018}. These studies use different datasets.
While ~\cite{bossard_2012} focuses on upper-body clothing, ~\cite{alamsyah_2019} investigates headwear and accessories using Instagram images.
~\cite{maryamah_2023} utilizes data collected from five official websites of well-known fashion brands.
These approaches achieve strong performance in tasks such as classification, retrieval, and landmark detection, particularly on large datasets ~\cite{wang_2018, maryamah_2023}.
However, ~\cite{wang_2018} highlight the limitation that many models do not incorporate domain-specific knowledge.
To address this issue, ~\cite{wang_2018} propose a knowledge-guided fashion network that jointly performs landmark localization and classification.
A key component is the Fashion Grammar Model, which combines a Bidirectional Convolutional Recurrent Neural Network (BCRNN) with domain-specific knowledge.
Grammar models originate from the field of computer vision and enable the modeling of semantic relationships between objects and humans, as well as the structured decomposition of scenes.
Additionally, category-guided and landmark-aware attention mechanisms are employed.
The results show that incorporating domain-specific knowledge leads to better performance compared to isolated neural network approaches.


\textbf{Problem Definition.}
In this project, we address the task of \emph{fashion item recognition and attribute prediction} from images. Given an input image containing one or multiple fashion products, the goal can be described as the following three steps:
\begin{enumerate}
    \item detect and segment: the fashion items present in the image,
    \item classify the type: of each detected item (e.g., shirt, dress, trousers), and
    \item predict attributes: describing each item, such as pattern, sleeve length, neckline, fabric, or fit.
\end{enumerate}
Formally, let $x \in \mathbb{R}^{H \times W \times C}$ denote an input image. The model learns a function
\[
f_\theta(x) = (y_c, y_a),
\]
where $y_c$ represents the categorical class label(s) of fashion items and $y_a$ denotes a set of attribute predictions. The task therefore combines image classification with multi-label attribute recognition. In order to solve the defined problem above, deep learning approaches will be applied, which will be described more in detail in section~\ref{sec:method}, Methods.

\textbf{Evaluation Criteria.}
Model performance is evaluated using standard computer vision metrics. Classification accuracy and top-$k$ accuracy are used to measure category prediction performance, while attribute prediction is evaluated using multi-label metrics such as precision, recall, and F1-score. These metrics quantify both recognition correctness and descriptive capability.

In addition to quantitative evaluation on benchmark datasets, we plan to conduct a qualitative evaluation using self-collected images. Custom photographs of fashion items and clothing are captured under real-world conditions, including different lighting, backgrounds, and viewpoints, and are fed to the trained model. This evaluation assesses the model's ability to generalize beyond the training distribution and provides insight into its practical applicability in real-world scenarios.

\textbf{Project Objective.}
The objective of the project is to design and implement deep learning models which are capable of (accurately) recognize fashion items and clothing while generating meaningful attribute descriptions. By combining classification and attribute prediction within a unified framework, the systems proposed later in this document aim to improve automated fashion understanding and try provide richer semantic information beyond simple category labels.

\textbf{Optional Extension: Outfit Recommendation.}
If sufficient development time is available, the project extends the recognition system toward an \emph{outfit recommendation task}. The goal is to utilize the predicted fashion categories and attributes as structured input for generating compatible clothing combinations.

Formally, let
\[
z = (y_c, y_a)
\]
denote the structured description extracted from an image. The recommendation system learns a function
\[
g_\phi(z) \rightarrow \mathcal{O},
\]
where $\mathcal{O}$ represents a set of recommended fashion items forming a coherent outfit. The task can be interpreted as learning compatibility relationships between clothing items based on semantic attributes such as color harmony, style consistency, and garment type constraints.

The recommendation quality can be evaluated using ranking-based metrics such as Precision and Recall\footnote{Recall at $k$ measures the fraction of relevant items that appear within
the top-$k$ recommendations generated by the model. It evaluates whether
compatible fashion items are ranked highly enough to be presented to
the user.}, compatibility accuracy, or user-preference simulation metrics. This extension demonstrates how visual recognition outputs can serve as a foundation for higher-level decision-making systems in intelligent fashion applications.



\section{Dataset}

We train and evaluate our model using the DeepFashion2 dataset \cite{ge2019deepfashion2},
which contains 100K images and 801K clothing items across 13 categories.

The dataset includes commercial images taken in controlled environments and
consumer images captured in real-world conditions. Each clothing item is annotated
with a bounding box, category label, and pixel-level segmentation mask.

Our project focuses on instance segmentation, where the model predicts, for each
clothing item in an RGB image, a bounding box, category label, and segmentation mask.

\subsection{Attribute Dataset}
\label{sec:attribute_dataset}

Deepfashion2 does not include attribute labels beyond the 13 garment categories. To enable attribute prediction, we use a subset of the Category and Attribute Prediction Benchmark from DeepFashion~\cite{liu2016deepfashion}.

This subset contains 20{,}000 images annotated with 26 attributes organized into six mutually exclusive groups:
\begin{itemize}
    \item \textbf{Pattern} (7): floral, graphic, striped, embroidered, pleated, solid, lattice
    \item \textbf{Sleeve length} (3): long sleeve, short sleeve, sleeveless
    \item \textbf{Garment length} (3): maxi, mini, no dress
    \item \textbf{Neckline} (4): crew, v-neckline, square, no neckline
    \item \textbf{Fabric} (6): denim, chiffon, cotton, leather, faux, knit
    \item \textbf{Fit} (3): tight, loose, conventional
\end{itemize}

Within each group, exactly one attribute is active per image, making each group a single-label classification problem. DeepFashion2 trains the detection and segmentation stage, while the DeepFashion subset trains the attribute classifier.


\section{Method}
\label{sec:method}

This section presents the methodological framework used for the fashion item recognition and attribute prediction task. The goal is to develop models that extract feature information from images and generate outputs consisting of item categories and visual attributes.

Two deep learning approaches are investigated. First, a transfer learning strategy is used, where a pre-trained convolutional neural network is adapted to the fashion domain and fine-tuned for the classification tasks. Second, a custom convolutional neural network (CNN) is trained from scratch to learn visual representations without pre-trained knowledge. The goal is then to compare the performance which the different models have achieve. 

In addition to the classification models, a pre-trained object detection model is used as a preprocessing step to localize fashion items within full images. The detector identifies bounding boxes around clothing items, which are then used to crop the corresponding regions. These cropped regions serve as inputs for the classification stage and for the task of attribute prediction.

The classification is performed using both the custom CNN and the EfficientNet-based transfer learning models. This pipeline ensures that the classifier focuses on the relevant clothing regions rather than background information, improving the quality of feature learning and final predictions. The prediction of the attributes follows the same approach. 

Both approaches are trained under comparable conditions for classification and attribute prediction. This setup enables a direct comparison of transfer learning and a custom-designed CNN in terms of performance and generalization ability.

This section presents the methodic framework used to address the task of fashion item recognition and attribute prediction. The objective is, as to create and train models that extract semantic information from images and generate structured descriptions consisting of item categories and visual attributes.

Rather than focusing on a single modeling strategy, this project investigates two different deep learning paradigms for visual understanding. The comparison aims to analyze how different principles of learning influence recognition performance and attribute prediction quality.

Both approaches considered in this work are trained under comparable conditions and optimized to learn category recognition and attribute estimation.

\subsection{Custom Convolutional Neural Network}

In addition to transfer learning, a custom convolutional neural network (CNN) was developed and trained from scratch for fashion item recognition. The model follows a residual learning paradigm inspired by ResNet architectures, enabling stable training of deeper networks through improved gradient propagation.

\subsubsection{Model Architecture}
\label{sec:cnn_rnn_architecture}

\paragraph{Residual Blocks}

The network is built from residual blocks consisting of a convolutional transformation path and a skip connection. Each block contains three consecutive $3\times3$ convolutional layers with Batch Normalization and ReLU activation, where the final convolution is applied without activation before residual addition:

\[
y = \mathrm{ReLU}(F(x) + S(x)).
\]

If input and output dimensions differ, the skip connection uses a $1\times1$ convolution with Batch Normalization; otherwise, an identity mapping is applied.

\paragraph{Network Structure}

The architecture comprises nine residual blocks with progressively increasing channel depth and spatial downsampling:

\begin{itemize}
    \item Block 1 initializes feature extraction.
    \item Block 2 applies residual learning followed by max pooling ($224 \rightarrow 112$).
    \item Blocks 3--4 expand channels to $2\times$ hidden units ($112 \rightarrow 56$).
    \item Blocks 5--6 increase capacity to $4\times$ hidden units ($56 \rightarrow 28$).
    \item Block 7 performs deep feature extraction ($8\times$ hidden units).
    \item Block 8 expands to $16\times$ hidden units with pooling ($28 \rightarrow 14$).
    \item Block 9 projects features to 320 channels and reduces resolution to $7\times7$.
\end{itemize}

\paragraph{Classification Head}

The classification head converts feature maps into predictions using:

\begin{itemize}
    \item Adaptive average pooling to $1\times1$,
    \item Flattening,
    \item Dropout ($p=0.4$),
    \item Fully connected output layer.
\end{itemize}

\paragraph{Design Rationale}

Residual connections improve gradient flow, Batch Normalization stabilizes training, and progressive channel expansion enables hierarchical feature learning. Max pooling reduces computational cost, while adaptive pooling and dropout enhance robustness and generalization.

Overall, the model combines deep residual feature extraction with a lightweight classifier for efficient fashion item classification.

\subsection{Transfer Learning with Efficient Net B1 and Efficient Net B2}
The next section describes briefly how the transfer learning approach was conducted to classify image labels and predict clothing attributes. 

\subsubsection{Architecture Overview}

For the transfer learning approach, EfficientNet-B1 and EfficientNet-B2 were used as backbone networks for feature extraction and classification. EfficientNet models are convolutional neural networks that apply a compound scaling strategy, jointly scaling depth, width, and input resolution to achieve high performance with strong parameter efficiency.

Both architectures consist of a convolutional stem followed by multiple stages of MBConv blocks, which combine depthwise separable convolutions, squeeze-and-excitation modules, and residual connections to efficiently extract hierarchical visual features.

EfficientNet-B2 is a scaled-up version of EfficientNet-B1, using increased depth, width, and input resolution, which allows for more expressive feature learning at the cost of slightly higher computational requirements.

EfficientNet was chosen due to its strong balance between accuracy and efficiency. Compared to larger standard CNN architectures, EfficientNet models are relatively lightweight and computationally efficient, which enables faster training and fine-tuning while still maintaining stable classification performance.

In this work, both models were initialized with pre-trained weights and fine-tuned for fashion item classification and attribute prediction. The original classification head was replaced with a task-specific fully connected layer matching the number of target categories and attributes.

\subsection{Training Setup}

This section describes the basic training configuration used for the fashion item classification and attribute prediction models. The same optimization strategy was applied to both the transfer learning models and the custom CNN architecture to ensure comparable experimental conditions.

\paragraph{Image Size}
All Images were scaled to a size of 224 x 224 pixels to allow a efficient training process. 

\paragraph{Dataset Balancing}

To the train the image classification models on DeepFashion2, a dedicated training sub set was constructed with an approximately uniform class distribution. For each clothing category, a similar number of images was selected in order to reduce class imbalance. This balancing strategy prevents the model from biasing predictions toward frequently occurring classes and encourages equal learning across all categories. As a result, the classifier learns more representative decision boundaries and improves overall generalization performance.

\subsubsection{Hyperparameters}

All models were optimized using adaptive gradient-based methods. The classification networks were trained with the Adam optimizer, while the Faster R-CNN ResNet50 FPN v2 detector used AdamW with a learning rate of $1\times10^{-4}$. These optimizers combine adaptive learning rates with momentum estimation, enabling stable convergence for deep neural networks.

For classification training, an exponentially decaying learning rate schedule ($\gamma = 0.95$) was applied. Training started with a learning rate of $0.001$, which was reduced after each epoch to allow rapid early learning followed by finer parameter updates. The Faster R-CNN ResNet50 FPN v2 model was fine-tuned on the DeepFashion2 dataset containing 13 clothing categories and one background class for 10 epochs using 3000 training images. Training progress was monitored using the total loss and box regression loss, reflecting both classification and bounding box refinement quality.

Classification models were trained using the CrossEntropyLoss function for multi-class prediction, while the detection model employed the standard Faster R-CNN objective combining classification and bounding box regression losses.

The number of training epochs varied between 5 and 20 depending on the experiment and model configuration. Shorter runs enabled rapid hyperparameter exploration, whereas longer training schedules improved convergence and overall classification and box prediction performance.

\subsection{Implementation Details}
\label{sec:implementation_details}

This section describes practical implementation decisions that influenced the training procedure and performance of both modeling approaches: the custom CNN architecture and the transfer learning models based on EfficientNet-B1 and EfficientNet-B2. To ensure a fair comparison, identical preprocessing, augmentation, and regularization strategies were applied whenever possible.

\paragraph{Data Augmentation}

Data augmentation was employed during training to increase data variability and reduce overfitting for all models. Both the custom CNN and the transfer learning approaches are prone to memorizing training samples when trained on fashion datasets with visually similar items. Augmentation therefore served as an important regularization technique.

Moderate geometric and photometric transformations were applied, including random horizontal flipping, small rotations, and slight brightness and contrast adjustments. These transformations simulate realistic variations in fashion images such as pose changes, illumination differences, and viewpoint variations while preserving semantic garment information.

A key practical consideration was the strength of augmentation. Strong augmentations can significantly increase task difficulty and hinder learning, particularly during fine-tuning of pre-trained networks such as EfficientNet-B1 and EfficientNet-B2. Therefore, augmentation parameters were carefully selected to maintain a balance between improving robustness and allowing efficient feature learning.

\paragraph{Transfer Learning Strategy (EfficientNet-B1 and EfficientNet-B2)}

For the EfficientNet-based models, a staged fine-tuning strategy was applied. Initially, the pre-trained backbone networks were frozen, and only the newly added classification head was trained. This step allowed the classifier to adapt to the fashion-specific label space while preserving the general visual features learned during pretraining.

In a second stage, fine-tuning was gradually extended to deeper layers of the network. Instead of updating all layers simultaneously, parts of the backbone were progressively unfrozen and jointly trained with the classifier. This step-wise fine-tuning strategy improved stability during training and helped the model adapt more effectively to domain-specific patterns in fashion images.

Empirically, this gradual unfreezing strategy led to improved test accuracy compared to training only the classifier head, as it allowed the model to refine higher-level feature representations without disrupting the already well-generalized low-level features.

\paragraph{Influence on Overfitting and Accuracy}

Experiments revealed a trade-off between augmentation and classification accuracy. Training without augmentation occasionally resulted in higher test accuracy values. However, models trained without augmentation showed a stronger tendency toward overfitting, characterized by rapid optimization on the training data and reduced generalization stability.

When augmentation was applied, overfitting effects were reduced and validation performance became more consistent across training epochs. Although peak accuracy sometimes decreased slightly, the models learned more robust and invariant feature representations. This behavior was observed consistently for both the custom CNN and the transfer learning models.

\paragraph{Regularization Techniques}

Additional regularization was introduced through dropout layers in the classification head of all classification models, helping to prevent overfitting during training. Batch normalization layers within the custom CNN stabilized gradient flow and improved convergence, while the pre-trained EfficientNet architectures inherently benefited from normalization and regularization mechanisms learned during large-scale pretraining.

\subsection{Object Detection}
For object detection, a Faster R-CNN ResNet50 FPN v2 model was used, which was pre-trained on the COCO dataset and subsequently fine-tuned on DeepFashion2.  
The model architecture combines a convolutional backbone for feature map extraction with a Region Proposal Network (RPN), which generates multiple anchor boxes at each spatial location on the feature maps. For each anchor box, the RPN predicts whether it corresponds to an object (foreground) or background.  
Object classification is performed based on the region proposals generated by the RPN. Each proposal is processed by a network to determine the object category, and the bounding boxes are subsequently refined to better fit the objects.

\textbf{Inference:}  
During inference, a large number of region proposals are generated, but only those with the highest classification scores are further processed. After final classification, overlapping and redundant bounding boxes are removed so that only the most likely and precise detections remain in the final output.

\textbf{Training:}  
Faster R-CNN ResNet50 FPN v2 was fine-tuned on the DeepFashion2 dataset with 13 clothing categories and 1 background category (AdamW, lr = 0.0001, 10 epochs, 3000 training images). During training, the total loss and box loss are monitored, where the box loss reflects the quality of both the region proposals and the refined bounding boxes.

\textbf{Evaluation:}  
Evaluation is performed using Mean Average Precision (mAP) and Mean Average Recall (mAR) using torchmetrics at an Intersection over Union (IoU) threshold of 0.75.  
Precision is defined as

\begin{equation}
    Precision = \dfrac{TP}{TP+FP}
\end{equation}

and describes the proportion of correctly predicted boxes among all predictions, while Recall is defined as

\begin{equation}
    Recall = \dfrac{TP}{TP+FN}
\end{equation}

and describes the proportion of ground-truth objects that were correctly detected.  
Two boxes are considered a match if their overlap is at least 75$\%$. This strict threshold was chosen to only count precise localizations as correct. As shown in Figure \ref{fig:train_res_obj_detection}, the box loss is low; however, the mAP remains relatively low, which is attributed to the large number of generated boxes per object.

\section{Pipeline}
The pipeline combines object detection, classification, and attribute prediction.  

\subsubsection{Object Detection}
The Faster R-CNN model outputs all detected bounding boxes together with a confidence score. Boxes with a score below a predefined threshold (0.5) are first removed. Subsequently, Non-Maximum Suppression (NMS) is applied class-wise: if two boxes of the same label overlap significantly (IoU > 0.5), only the box with the higher score is retained. This prevents multiple detections of the same clothing item.

\subsection{Pipeline Evaluation}

The test set consists of 5000 images from DeepFashion2. Since ground truth annotations are available only for bounding boxes and categories, the evaluation is limited to detection and classification performance. The goal is to measure how well the pipeline detects the correct number of garments and classifies them correctly. Three metrics are used:

\begin{itemize}
    \item \textbf{Exact Match} 1 if the number of predicted boxes matches the ground truth and all predicted labels are contained in the ground truth labels, otherwise 0.
    \item \textbf{Coverage} Fraction of ground truth labels covered by predictions: $\frac{\text{matched labels}}{\text{GT labels}}$.
    \item \textbf{Precision} Penalty term for over-prediction: $
    \frac{\text{GT labels}}{\text{predicted boxes}}$  (only applied when predictions > GT, otherwise 1.0).
\end{itemize}
